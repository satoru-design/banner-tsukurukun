## 1. サービス概要

### 1.1 名称
**勝ちバナー作る君**（旧称: Ironclad Banner Studio）

### 1.2 サービス本質
ブリーフ（商材/ターゲット/目的）を入力するだけで、AI が**勝ちパターンを学習した広告バナー**を**17サイズ一括生成**する SaaS ツール。

### 1.3 既存ツールとの本質差別化（独自価値）

| 軸 | Canva / Figma | **勝ちバナー作る君** |
|---|---|---|
| 制作プロセス | テンプレ設計 → スロット埋め（2フェーズ） | ブリーフ → 完成（**1フェーズ**） |
| サイズ展開 | サイズごとに手動レイアウト調整 | **17サイズ統一感を AI が自動配置** |
| 過去の成功体験 | 個人の勘・記憶頼み | **勝ちバナーを AI 解析 → 次回反映** |
| 商品画像保持 | 手動配置 | **改変禁止モードで素材精度100%** |

**→ コア差別化メッセージ**: 「**テンプレを作る時間、もう要りません**」

### 1.4 ターゲット顧客

| セグメント | 規模感 | 想定プラン |
|---|---|---|
| 個人マーケター・副業 | 月広告予算 1万〜30万 | スターター ¥3,980 |
| 年商10億円以下EC事業者 | 月広告予算 50万〜500万 | プロ ¥14,800 |
| 大手・広告代理店 | 月広告予算 1000万〜 | Plan C（個別契約） |

### 2.2 4セッション目グレーアウト仕様
フリープラン超過時の挙動：
- 通常通り API 実行（コスト ¥50/セッション）
- 出力画像に**全面ウォーターマーク**「PREVIEW ONLY - UPGRADE TO DOWNLOAD」
- CSS filter: `grayscale(80%) opacity(70%)` で視覚的劣化
- DLボタンを「アップグレード」CTA に置換
- 何度試しても綺麗な版は手に入らない設計

**ROI試算**: 1セッション原価¥250 vs 1人プロ転換LTV ¥177,600 → **約700倍のROI**

## 3. クーポン / プロモーション戦略

### 3.1 Stripe Promotion Codes 活用

#### 初回キャンペーン: 「FRIENDS」初月無料
```
Coupon: 100% off, duration=once
Promotion Code: "FRIENDS"
適用範囲: Pro プランのみ
Max Redemptions: 100名先着
Per-Customer: 1回（Stripe Customer 単位）
有効期限: ローンチ後3ヶ月
```

**FB/IG 告知例:**
> 🎉 Beta公開記念
> 友達限定キャンペーン
> コード『FRIENDS』入力で初月無料（Pro ¥14,800 → ¥0）
> **先着100名様**

#### 段階的特典バリエーション
- **3ヶ月50%OFF**: `EARLYBIRD3` (duration=repeating, months=3)
- **永年30%OFF初期サポーター**: `FOUNDING50` (duration=forever, max=50)
- **アンバサダー個別**: `KOIKE-XXXX` 形式で紹介者ごと発行

### 3.2 リスク対策
- `Per-Customer: 1` で同人連打防止
- Google SSO 必須でメアド変更による再利用防止
- Pro プラン限定で適用 → 高単価+本気顧客のみに絞り込み

**→ 健全な PMF 達成水準**。プロ転換10%でこの数字。勝ちバナー独自機能で15-20%狙える可能性。

## 5. アーキテクチャ判断

### 5.1 認証
- **NextAuth.js** で Google SSO 統一
- 既存 Basic Auth は廃止
- 自分（小池）も Google アカウント `str.kk.co@gmail.com` でログイン → admin plan
- 顧客と同じ目線で UX 確認可能

### 5.2 ドメイン構成
- **アプリ**: `autobanner.jp/`（既存・継続）
- **LP**: `autobanner.jp/lp01`, `/lp02` ...（A/Bテスト前提のサブパス構成）
- 新規ドメイン取得不要 / DNS変更不要
- middleware で `/lp**` は無認証、`/` 以下のアプリは Google SSO 必須

### 5.3 課金エンジン
- **Stripe** 採用
- Stripe Checkout でサブスク購入
- Stripe Customer Portal で解約・支払い変更（顧客自己完結）
- Stripe Webhook で `subscription.created/updated/deleted` を受けて DB の plan 同期
- メータード課金（Pro超過分 ¥80/回）も Stripe 標準機能で実装

### 5.4 問い合わせフォーム
- 既存 Formspree (`xaqaazaz`) 流用 もしくは新規発行（free tier）
- LP の Plan C 問い合わせ動線で使用

## 10. 完了の定義（PMF達成判断）

以下が満たされたら「PMF 達成」と判定：

1. 月次新規獲得: **連続3ヶ月で 100人以上**
2. 有料転換率: **連続3ヶ月で 25%以上**
3. 月次解約率: **5%以下**（健全 SaaS 水準）
4. NPS（推奨度）: **40以上**（業界平均超え）
5. 「Pro プラン」連続契約者: **30名以上**

達成時の戦略選択肢：
- A. 機能拡張で ARPU 向上（業種テンプレ・チーム機能等）
- B. 海外展開（英語UI・USD課金）
- C. M&A 交渉開始（広告制作系SaaSが買い手候補）
