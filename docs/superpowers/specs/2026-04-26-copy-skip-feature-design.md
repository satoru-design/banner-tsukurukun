# Phase A.9: コピースキップ機能 設計書

**作成日:** 2026-04-26
**ステータス:** ブレインストーミング完了 / 実装プラン作成前
**前提:** Phase A.8（勝ちバナー参照機能）本番動作確認済み
**配置場所:** `docs/superpowers/specs/2026-04-26-copy-skip-feature-design.md`

---

## 1. 背景・目的

現状、STEP 2 のサジェスト選択で 4 つのコピースロット（メイン / サブ / ターゲット / 権威）と CTA はすべて生成バナーに必ず含まれる。実運用では「サブコピーは要らない」「権威バッジを外したい」「CTA ボタンなしでブランディング寄せ」等のニーズがあり、現状はダミー文字を入れる以外に回避策がない。

各スロットに「使う / 使わない」トグルを追加し、不使用時はそのテキストを生成バナーから完全に除外できるようにする。

---

## 2. 設計判断の根本原則

### 2.1 メインコピーは必須を維持
バナーの主軸メッセージなので、空にできてしまうとほぼ画像のみのスカスカ素材が生成される。Phase A.9 ではメインコピー（copies[0]）はスキップ不可とする。

### 2.2 バックエンド変更ゼロ
既存の `buildBakeTextInstruction`（[prompt-helpers.ts:9-64](claude_pjt/banner-tsukurukun/src/lib/image-providers/prompt-helpers.ts:9)）は各テキストを `if (sub)` `if (cta)` 等で条件分岐済み。**空文字を渡せば bake 指示が出ない設計に既になっている**ため、フロント側で空文字を送るだけで実装完了。

### 2.3 既存挙動の完全互換
全スロット ON（デフォルト）状態では、Phase A.8 完了時点と**1ピクセルも変わらない**結果が出ることを保証する。

---

## 3. 機能要件

### 3.1 ユーザーストーリー
> マーケターとして、生成するバナーに**特定のコピー要素を含めたくない**ことがある（例: 「権威バッジは要らない」「CTAなしでブランディング寄せ」）。STEP 2 で各スロットに「☑ このスロットを使う」トグルを設置し、OFF にすればそのテキストは生成バナーに描画されない。

### 3.2 スコープ

| スロット | スキップ可？ | 理由 |
|---|---|---|
| コピー1（メイン） | ❌ 必須 | バナーの主軸メッセージ |
| **コピー2（サブ）** | ✅ 可 | 補助的、なくてもバナー成立 |
| **コピー3（ターゲット/価格訴求）** | ✅ 可 | 配信側で別途設定可 |
| **コピー4（権威/ダメ押し）** | ✅ 可 | バッジは任意要素 |
| **CTA** | ✅ 可 | ブランディング重視時に外したいニーズ |
| デザイン要件1〜4 | ❌ スコープ外 | 設計指示は常に必要 |
| トーン | ❌ スコープ外 | 創作方向性は常に必要 |
| 注意事項 | ❌ スコープ外 | 法務観点 |

### 3.3 スコープ外（Phase 2 以降）
- メインコピーのスキップ
- デザイン要件のスキップ
- トーン / 注意事項のスキップ
- スロット ON/OFF のプリセット保存

## 4. UI 設計（Pattern B：トグル式）

### 4.1 ON 状態（デフォルト）
```
┌────────────────────────────────────────────┐
│ コピー2（サブ）        [☑ このスロットを使う] │
├────────────────────────────────────────────┤
│ [A] あの頃の軽やかさ、諦めないで。           │
│ [B] 「なんだか体が重い…」そのサイン、内側… │
│ [C] 今だけの初回限定キャンペーン中           │
│ [D] 満足度92%※の実力派                      │
│                                             │
│ 自由入力で上書き                            │
│ [_____________________________________]     │
└────────────────────────────────────────────┘
```
既存 SuggestField とほぼ同じ。違いは右上にトグル追加のみ。

### 4.2 OFF 状態（不使用）
```
┌────────────────────────────────────────────┐
│ コピー2（サブ）        [☐ このスロットを使う] │
├────────────────────────────────────────────┤
│   このスロットは生成バナーに含めません       │
│   （候補は disabled、選択不可）              │
└────────────────────────────────────────────┘
```
- フィールド全体を視覚的にグレーアウト（opacity-40 等）
- 候補ボタンは disabled 化
- 自由入力欄も disabled
- 内部的に選択値は空文字 `''`

### 4.3 トグル操作仕様
- クリック1回で ON⇄OFF 切替
- ON→OFF: 選択値を空文字に強制リセット
- OFF→ON: 空のまま（ユーザーが新たに候補選択 or 自由入力）
- メインコピー / デザイン要件 / トーン / 注意事項にはトグル表示しない

### 4.4 「次へ」ボタン制御
- `canProceed` ロジック更新:
  - メインコピー（copies[0]）必須（既存と同じ）
  - デザイン要件1（designRequirements[0]）必須（既存と同じ）
  - トーン必須（既存と同じ）
  - **ON状態の copies[1〜3] / cta は必須**（OFF状態のものは不要）

## 5. データモデル

### 5.1 State（Frontend のみ）

`IroncladSuggestSelector` 内に追加:
```typescript
interface EnabledSlots {
  sub: boolean;       // copies[1]
  target: boolean;    // copies[2]
  authority: boolean; // copies[3]
  cta: boolean;
}

const [enabledSlots, setEnabledSlots] = useState<EnabledSlots>({
  sub: true,
  target: true,
  authority: true,
  cta: true,
});
```

### 5.2 永続化なし
Phase 1 はセッションスコープ。STEP 1 戻る or リロードで全 ON にリセット。
将来「前回設定を覚えておきたい」となったら localStorage 化を検討。

### 5.3 バックエンド送信値
- スロット ON: ユーザー選択値（既存通り）
- スロット OFF: 空文字 `''`

```typescript
// onNext に渡す materials の例（CTA OFF, 権威 OFF）
{
  copies: [
    'メイン本文',           // [0] 必須・常に値
    'サブ本文',             // [1] ON
    'ターゲット本文',       // [2] ON
    '',                     // [3] OFF → 空文字
  ],
  cta: '',                  // OFF → 空文字
  ...
}
```

## 7. エッジケース

| ケース | 期待挙動 | 対応箇所 |
|---|---|---|
| ON→OFF: 候補選択済 | 選択値が空文字に強制リセット | `setEnabledSlots` の onChange ハンドラ内 |
| OFF→ON: 空のまま | 空のまま、ユーザーが選び直す or 入力 | 復元しない |
| 全スキップ可スロット OFF | メイン+デザイン要件+トーンのみで生成 → 「DETOX」「ALL」程度の超ミニマル広告 | UI上「次へ」は通る、生成APIも空文字許容 |
| 候補未選択 + トグルOFF | OFF優先、空文字で送信、`canProceed` は ON スロットのみ評価 | `canProceed` ロジック |
| ON状態で候補未選択 | 既存通り「次へ」ボタン disabled | 既存 `canProceed` |

## 9. テスト戦略

### 9.1 手動E2Eテスト
1. **デフォルト動作（全ON）**: STEP 1 → STEP 2 → 全スロット選択 → STEP 3 生成 → Phase A.8 と同じバナー出力
2. **CTA OFF**: CTA トグルだけOFF → 生成 → CTAボタンが描画されないことを確認
3. **権威バッジ OFF**: copies[3] OFF → 生成 → 右上の権威バッジが描画されないことを確認
4. **複数 OFF**: サブ + ターゲット + CTA OFF → 生成 → メインコピーと権威バッジ程度のミニマル
5. **トグル OFF→ON**: 一度 OFF にしてから ON に戻す → 候補選択UIが復活、選択値は空のまま
6. **OFF状態で「次へ」**: ON スロットを全て選択 → 「次へ」ボタン有効 → STEP 3 遷移成功

### 9.2 既存機能の互換性確認
- Phase A.8 の勝ちバナー参照ON状態でも上記E2Eが通ること
- LP URL 自動抽出 + コピースキップの組み合わせも問題なし
